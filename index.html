General Configuration

Connectable is RAM-only and resets to ON at every power-up. Turn OFF to ship a pure non-connectable beacon; power-cycle the device to make it connectable again.
The BLE address is fixed (the chip's factory FICR address) and cannot be changed.

Slot Configuration

1. Open nRF Connect and place the phone right next to the beacon (~0 m). 2. Read this device's RSSI (dBm). 3. Type that value straight into the Measured Power @0m field above, then click Write slot. The firmware converts automatically: iBeacon subtracts 41 dB (0 m to 1 m reference), Eddystone uses the 0 m value directly.
Encodes scheme + suffix per the Eddystone spec, max 17 bytes after the scheme.
TLM is auto-generated: VDD voltage, die temperature, advertising counter, uptime.
AD type 0xFF (Manufacturer Specific Data). Default company 0x5054 = Tapama.
Advanced: you build the entire AD yourself. Malformed lengths mean the slot will not advertise. Spaces are ignored.

Est. remaining-
Voltage is read live from the device (chip VDD). Level maps 3.0 V = 100%, 2.0 V = 0% (3 V lithium). Remaining time = (level x capacity) / estimated current, a rough figure only. Assumes the cell powers the nRF directly. For exact numbers use a power profiler.

Config Profile (Clone / Provision)

Export the full configuration (general + all 6 slots) as a portable profile, then import it onto other devices for fleet provisioning. The password is NOT included; each device keeps its own.
